Technology

openFOIA is a Python/Django application. Many of the scripts written to collect data are also written in Python.

API

All of the information presented is available through an API. The API documentation provides more details.

Bulk Data

The data source for the API and the tool is a set of YAML files that contains all the data. These files are the canonical source for our data and are updated through a combination of automated and manual processes.

  • YAML data files

Repositories

  • foia Our main repository (also contains the bulk data)
  • foia-hub The repository for openFOIA.

Data Sources

We use multiple sources for the information we display about each government component.

Our contact and processing times data come from both foia.gov and manually culling government FOIA web pages. Descriptions and acronyms come from the USAGOV Contacts API. We populate keyword search terms using the Federal Register API. Finally, the FOIA library links are scraped from each agency's FOIA web page where relevant.

Our source code provides more information and details on how each data source is processed.
